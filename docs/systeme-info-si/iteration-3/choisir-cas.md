# Cas d'étude : mairie

## Choix du cas

Le cas choisi est une **mairie**.

Ce cas est intéressant car une mairie utilise un système d'information assez varié, mais compréhensible :

- accueil du public ;
- gestion administrative ;
- état civil ;
- démarches en ligne ;
- messagerie ;
- stockage de documents ;
- relation avec d'autres organismes publics.

Une mairie manipule aussi des données sensibles : identité des habitants, documents administratifs, demandes sociales, inscriptions scolaires ou données liées aux agents municipaux.

## Organisation

L'organisation étudiée est une **mairie de taille moyenne**.

Elle regroupe plusieurs services :

- accueil et état civil ;
- urbanisme ;
- ressources humaines ;
- finances ;
- services scolaires ;
- services techniques ;
- communication ;
- direction générale.

## Activité principale

La mairie assure des services publics de proximité pour les habitants.

Ses activités principales sont :

- recevoir et traiter les demandes des citoyens ;
- gérer les actes d'état civil ;
- organiser les services municipaux ;
- gérer les inscriptions scolaires ou périscolaires ;
- suivre les demandes d'urbanisme ;
- communiquer avec les habitants ;
- administrer les agents, les budgets et les équipements de la commune.

## Périmètre du SI étudié

Le périmètre retenu est le **SI administratif de la mairie**.

Il comprend :

- les postes informatiques des agents ;
- les applications métier utilisées par les services ;
- la messagerie professionnelle ;
- le stockage et le partage de documents ;
- l'accès internet ;
- les sauvegardes ;
- les comptes utilisateurs ;
- les accès aux plateformes administratives externes.

Le périmètre ne détaille pas les systèmes très spécialisés comme la vidéosurveillance, la téléphonie avancée ou les équipements industriels des services techniques.

## Informations connues

### Outils utilisés

| Outil | Utilisation probable |
| --- | --- |
| Postes Windows | travail quotidien des agents |
| Suite bureautique | rédaction de courriers, tableaux, documents administratifs |
| Messagerie professionnelle | échanges internes, échanges avec citoyens et partenaires |
| Logiciel d'état civil | actes de naissance, mariage, décès, demandes administratives |
| Logiciel finances / comptabilité | budget, factures, dépenses, recettes |
| Logiciel RH | gestion des agents, congés, paie ou dossiers du personnel |
| Logiciel urbanisme | suivi des permis de construire et déclarations préalables |
| Serveur de fichiers ou cloud | stockage et partage de documents |
| Sauvegarde | récupération des données en cas de panne ou d'erreur |
| Site web de la mairie | informations publiques et démarches en ligne |
| Portails administratifs externes | échanges avec préfecture, services publics ou organismes partenaires |

### Utilisateurs

| Utilisateur | Besoin principal |
| --- | --- |
| Agents d'accueil | consulter les informations, enregistrer les demandes, orienter le public |
| Agents d'état civil | produire et consulter des actes administratifs |
| Service urbanisme | suivre les dossiers liés aux travaux et autorisations |
| Service finances | gérer les budgets, factures et paiements |
| Service RH | gérer les dossiers des agents municipaux |
| Services techniques | accéder aux demandes d'intervention et documents utiles |
| Direction générale | piloter l'activité et accéder aux informations de synthèse |
| Élus | consulter certains documents, suivre les décisions et projets |
| Administrateur informatique ou prestataire | gérer les comptes, postes, sauvegardes, sécurité et incidents |
| Habitants | accéder aux services, déposer des demandes, recevoir des réponses |

### Contraintes

| Contrainte | Explication |
| --- | --- |
| Confidentialité | les données personnelles des habitants et agents doivent être protégées |
| Disponibilité | l'accueil, l'état civil et les démarches importantes doivent rester accessibles |
| Sécurité des accès | chaque agent ne doit accéder qu'aux informations nécessaires à son service |
| Sauvegarde | les documents administratifs et données métier ne doivent pas être perdus |
| Réglementation | la mairie doit respecter le RGPD et les obligations liées aux données publiques |
| Continuité de service | certaines démarches ne peuvent pas être interrompues trop longtemps |
| Budget limité | les choix techniques doivent rester adaptés aux moyens d'une collectivité |
| Utilisateurs variés | tous les agents n'ont pas le même niveau informatique |
| Dépendance aux prestataires | certaines applications métier peuvent être hébergées ou maintenues par des fournisseurs externes |
