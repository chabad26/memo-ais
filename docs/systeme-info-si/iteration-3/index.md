# Analyse d'une architecture SI

Cette partie sert à apprendre à lire et analyser une architecture de système d'information.

L'objectif est de ne pas seulement décrire les composants, mais de comprendre :

- le rôle de chaque élément ;
- les liens entre les applications, les serveurs, le réseau et les utilisateurs ;
- les dépendances importantes ;
- les points faibles possibles ;
- les choix techniques qui améliorent la disponibilité, la sécurité ou la maintenance.

Une analyse d'architecture SI permet de repérer ce qui fonctionne bien, ce qui peut devenir fragile, et ce qu'il faut surveiller en priorité.
