# Intro AIS

## Introduction au métier d'AIS

AIS signifie généralement **Administrateur d'Infrastructures Sécurisées**.

Le rôle d'un AIS est de comprendre, administrer et sécuriser des systèmes informatiques.

Cela demande des bases solides dans plusieurs domaines :

- le matériel informatique,
- les systèmes d'exploitation,
- Linux,
- le terminal,
- le réseau,
- la cybersécurité.

L'objectif de cette introduction est de prendre en main ces notions progressivement, avec des explications simples et des exemples concrets.

## Ce que l'on va découvrir

### Matériel informatique

Avant de parler système ou réseau, il faut comprendre les composants d'une machine.

On y retrouve notamment :

- le processeur,
- la mémoire RAM,
- le stockage,
- la carte mère,
- la carte réseau,
- les périphériques.

Ces éléments permettent de comprendre sur quoi repose un ordinateur ou un serveur.

### Système d'exploitation et Linux

Un système d'exploitation sert d'intermédiaire entre l'utilisateur, les logiciels et le matériel.

Linux est très utilisé dans le monde professionnel, surtout côté serveur.

Les notions importantes sont :

- le kernel,
- les distributions Linux,
- l'arborescence des fichiers,
- les utilisateurs et les droits,
- les processus et les services,
- les logs.

### Terminal et commandes Linux

Le terminal permet d'administrer une machine avec des commandes.

Il est indispensable pour :

- se déplacer dans les dossiers,
- consulter des fichiers,
- gérer les droits,
- vérifier l'état du système,
- diagnostiquer un problème,
- administrer un serveur à distance.

### Réseau

Le réseau permet aux machines de communiquer entre elles.

Les bases à connaître sont :

- l'adresse IP,
- le masque réseau,
- la passerelle,
- le DNS,
- le DHCP,
- les ports,
- les protocoles,
- la différence entre switch et routeur.

Ces notions sont essentielles pour comprendre comment une machine communique sur un réseau local ou sur Internet.

### Cybersécurité

La cybersécurité consiste à protéger les systèmes, les réseaux et les données.

Elle repose notamment sur :

- la limitation des accès,
- la gestion des droits,
- le chiffrement,
- la surveillance,
- la correction des failles,
- la réaction aux incidents.

Un AIS doit avoir les bons réflexes pour administrer sans exposer inutilement les systèmes.

## Progression conseillée

Pour avancer dans un ordre logique :

1. Commencer par le matériel.
2. Comprendre le rôle du système d'exploitation.
3. Découvrir Linux et son organisation.
4. Pratiquer les commandes de base dans le terminal.
5. Étudier les bases du réseau.
6. Relier ces notions à la cybersécurité.

## Compétences visées

À la fin de cette introduction, l'objectif est de savoir :

- expliquer le rôle des composants d'un ordinateur,
- comprendre ce qu'est un système d'exploitation,
- distinguer Linux, le kernel et une distribution,
- utiliser quelques commandes Linux utiles,
- comprendre les bases d'une configuration réseau,
- identifier les premiers réflexes de sécurité,
- lire des informations système simples pour diagnostiquer un problème.

## Réflexes importants

Dans le métier d'AIS, il faut apprendre à observer avant d'agir.

Avant de modifier une configuration, on cherche à comprendre :

- sur quelle machine on travaille,
- avec quel utilisateur,
- quels services sont actifs,
- quelle est la configuration réseau,
- quels messages apparaissent dans les logs,
- quels droits sont nécessaires.

!!! tip "Bon réflexe"
    Une bonne administration commence souvent par trois actions simples : observer, vérifier, puis agir.

## Résumé rapide

Un AIS travaille à la croisée de plusieurs domaines :

- **système** : comprendre et administrer les machines,
- **réseau** : permettre aux machines de communiquer,
- **sécurité** : protéger les accès et les données,
- **diagnostic** : chercher l'origine d'un problème,
- **documentation** : garder une trace claire des actions réalisées.

Cette section sert donc de base pour construire les réflexes techniques nécessaires à l'administration d'infrastructures sécurisées.
