# Glossaire - Intro AIS

## Termes à retenir

| Terme | Définition courte |
| --- | --- |
| AIS | Administrateur d'infrastructures sécurisées : profil orienté systèmes, réseaux, sécurité et documentation. |
| Matériel | Ensemble physique : processeur, mémoire, disque, carte réseau, périphériques. |
| Système d'exploitation | Logiciel central qui fait le lien entre matériel, applications et utilisateur. |
| Linux | Famille de systèmes d'exploitation libres très utilisée côté serveur. |
| Shell | Interface en ligne de commande permettant d'exécuter des commandes. |
| Réseau | Ensemble d'équipements capables d'échanger des données. |
| Adresse IP | Identifiant logique d'une machine sur un réseau. |
| Port | Numéro qui identifie un service réseau sur une machine. |
| Chiffrement | Transformation d'une donnée lisible en donnée illisible sans clé. |
| Hygiène numérique | Ensemble de réflexes simples pour réduire les risques : mises à jour, mots de passe, sauvegardes, vigilance. |

## Manipulations faites

| Manipulation | Commandes ou actions |
| --- | --- |
| Identifier le système | `uname -a`, `cat /etc/os-release` |
| Se repérer dans le terminal | `pwd`, `ls`, `cd`, `tree` |
| Lire un fichier | `cat`, `less`, `head`, `tail` |
| Vérifier le réseau | `ip addr`, `ip route`, `ping`, `ss -tulpen` |
| Comprendre une menace | Identifier actif, vulnérabilité, menace, impact et mesure de protection. |
