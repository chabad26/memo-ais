# Glossaire Réseaux sécurisés - Itération 6

## Sujet

Préparation défensive, validation de l'infrastructure et préparation au challenge final.

## Termes à retenir

| Terme | Définition courte |
| --- | --- |
| Challenge final | Mise en situation où l'infrastructure doit résister à des tests. |
| Validation défensive | Vérification que les protections attendues sont en place. |
| Surface d'exposition | Ensemble des services accessibles depuis un point donné. |
| Durcissement | Réduction des faiblesses de configuration. |
| Test croisé | Test réalisé depuis une autre machine ou un autre apprenant. |
| CTF | Challenge technique orienté résolution de failles ou objectifs. |
| Preuve de sécurité | Capture, log ou résultat montrant qu'une protection fonctionne. |

## Manipulations faites

| Manipulation | Commandes ou actions |
| --- | --- |
| Vérifier l'infrastructure | VLANs, pfSense, VPN, nftables, services exposés. |
| Tester depuis l'extérieur | Accès depuis un PC tiers, ports réellement visibles. |
| Corriger les écarts | Ajustement règles firewall, services, routes. |
| Préparer le CTF | Définir objectifs, risques, preuves attendues. |
| Documenter la défense | Tableau tests, résultats, corrections. |
