# Glossaire SI - Itération 2

## Sujet

Enjeux, risques, normes et priorisation dans un système d'information.

## Termes à retenir

| Terme | Définition courte |
| --- | --- |
| Enjeu | Ce que l'organisation cherche à préserver ou améliorer. |
| Risque | Possibilité qu'une menace exploite une vulnérabilité et produise un impact. |
| Menace | Cause potentielle d'incident : humaine, technique, naturelle ou organisationnelle. |
| Vulnérabilité | Faiblesse exploitable dans un système, un processus ou une configuration. |
| Impact | Conséquence métier, technique, financière, juridique ou humaine. |
| Probabilité | Estimation de la chance qu'un risque se réalise. |
| Priorisation | Classement des risques selon criticité, impact et probabilité. |
| Norme | Référentiel de bonnes pratiques ou d'exigences. |

## Manipulations faites

| Manipulation | Objectif |
| --- | --- |
| Identifier les risques | Relier actif, menace, vulnérabilité et impact. |
| Classer les risques | Prioriser les actions de mitigation. |
| Relier normes et sécurité | Comprendre l'intérêt d'un cadre commun. |
| Synthétiser | Produire une vision claire enjeux, risques et normes. |
