# Glossaire SI - Itération 3

## Sujet

Analyse d'une architecture SI, cartographie, usages, risques et propositions d'amélioration.

## Termes à retenir

| Terme | Définition courte |
| --- | --- |
| Cas d'étude | Situation réelle ou réaliste utilisée pour analyser un SI. |
| Usage | Manière concrète dont les utilisateurs utilisent le SI. |
| Architecture SI | Organisation des composants applicatifs, techniques et des flux. |
| Mesure d'amélioration | Action proposée pour réduire un risque ou améliorer la résilience. |
| Restitution | Présentation synthétique et argumentée d'une analyse. |
| Arbitrage | Choix entre plusieurs solutions selon coût, risque, complexité et impact. |

## Manipulations faites

| Manipulation | Objectif |
| --- | --- |
| Choisir un cas | Définir un périmètre analysable. |
| Cartographier | Produire une représentation claire des composants et flux. |
| Analyser les usages | Relier pratiques réelles et risques. |
| Proposer des améliorations | Prioriser des mesures réalistes et justifiées. |
| Préparer une restitution | Transformer l'analyse en discours compréhensible. |
