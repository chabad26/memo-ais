# Travail et données personnelles

Dans le cadre du travail, l'employeur traite régulièrement des données personnelles sur les salariés, agents, candidats, stagiaires ou intérimaires.

Le RGPD s'applique donc aussi à la vie professionnelle.

## Données concernées

Les données traitées au travail peuvent être très variées :

- identité,
- coordonnées,
- CV et parcours professionnel,
- contrat de travail,
- rémunération,
- absences,
- évaluations,
- données de paie,
- badge d'accès,
- logs de connexion,
- messagerie professionnelle,
- équipement informatique,
- géolocalisation de véhicules,
- images de vidéosurveillance.

Certaines données peuvent être sensibles, par exemple des données de santé ou des informations liées à une situation de handicap.

## Recrutement

Lors d'un recrutement, l'employeur ne doit collecter que les informations nécessaires pour évaluer la capacité du candidat à occuper le poste.

Exemples de données pertinentes :

- CV,
- expérience professionnelle,
- diplômes,
- compétences,
- coordonnées de contact.

Exemples de données à éviter si elles ne sont pas justifiées :

- situation familiale détaillée,
- opinions politiques,
- religion,
- données de santé,
- informations sans lien avec le poste.

## Gestion du personnel

Après l'embauche, l'employeur peut traiter des données pour gérer la relation de travail.

Exemples :

- établir la paie,
- gérer les congés,
- organiser les horaires,
- suivre les formations,
- gérer les accès aux locaux,
- fournir du matériel informatique,
- administrer les comptes utilisateurs,
- respecter des obligations légales.

Ces traitements doivent respecter les principes du RGPD : finalité, minimisation, sécurité, transparence et durée de conservation limitée.

## Surveillance et outils numériques

L'employeur peut mettre en place certains dispositifs de contrôle ou de sécurité, mais ils doivent être proportionnés et expliqués.

Exemples :

- contrôle d'accès par badge,
- vidéosurveillance,
- filtrage web,
- logs de connexion,
- suivi du temps de travail,
- géolocalisation de véhicules professionnels.

Ces outils ne doivent pas devenir une surveillance permanente et excessive des salariés ou agents.

!!! warning "Point de vigilance"
    Un outil technique installé pour la sécurité peut aussi collecter des données personnelles. Par exemple, un proxy web, un EDR, un système de badge ou des logs VPN peuvent révéler des habitudes de travail.

## Droit d'accès des salariés et agents

Un salarié ou agent peut demander à son employeur l'accès aux données personnelles qui le concernent.

Cela peut concerner :

- son dossier professionnel,
- certaines données RH,
- des informations d'évaluation,
- des données de badge,
- certains courriels professionnels,
- des traces liées à ses outils numériques.

L'employeur doit être capable de répondre dans le respect du RGPD, sans transmettre les données personnelles d'autres personnes.

## Télétravail

Le télétravail augmente les enjeux liés aux données personnelles et à la sécurité.

Points de vigilance :

- accès distant sécurisé,
- VPN ou solution équivalente,
- MFA,
- protection du poste de travail,
- confidentialité à domicile,
- gestion des documents imprimés,
- outils collaboratifs,
- séparation entre usages personnels et professionnels.

Pour un AIS, le télétravail implique aussi de bien gérer les journaux, les accès, les équipements, les sauvegardes et la révocation des comptes.

## Exemple concret

Une entreprise met en place un système de badge pour contrôler les accès aux locaux.

Ce traitement peut être légitime pour la sécurité et le suivi des horaires.

Mais l'entreprise doit :

- informer les salariés,
- limiter les personnes qui accèdent aux données,
- définir une durée de conservation,
- éviter une surveillance excessive,
- sécuriser le système,
- documenter le traitement dans le registre.

## Bons réflexes pour l'employeur

- collecter uniquement les données nécessaires,
- informer les salariés, agents et candidats,
- limiter les accès aux données RH,
- sécuriser les outils de travail,
- encadrer les dispositifs de contrôle,
- documenter les traitements dans le registre,
- prévoir des durées de conservation,
- répondre aux demandes d'accès,
- consulter le DPO en cas de doute.

## À retenir

- Les données personnelles existent aussi dans la relation de travail.
- L'employeur peut traiter des données, mais il doit respecter un cadre.
- Les outils techniques peuvent produire des traces personnelles.
- Le salarié ou agent conserve des droits sur ses données.
- Pour un AIS, les logs, accès, badges, postes, VPN, messagerie et sauvegardes sont des points sensibles.
