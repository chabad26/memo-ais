# RGPD en collectivité

Les collectivités territoriales traitent beaucoup de données personnelles dans leurs missions de service public.

Cela concerne les communes, départements, régions, intercommunalités, CCAS, établissements publics locaux et structures associées.

## Pourquoi les collectivités sont concernées ?

Une collectivité gère des services pour les habitants, les familles, les agents et les partenaires.

Elle traite donc des données personnelles dans de nombreux domaines :

- état civil,
- inscriptions scolaires et périscolaires,
- restauration scolaire,
- listes électorales,
- action sociale,
- logement,
- urbanisme,
- police municipale,
- vidéosurveillance,
- gestion des agents,
- contrôle d'accès,
- site web et démarches en ligne.

Le RGPD est important parce que ces données peuvent être sensibles ou concerner des personnes vulnérables : enfants, familles en difficulté, personnes âgées, personnes handicapées, agents publics.

## Responsable de traitement

Dans une collectivité, le **responsable de traitement** est généralement la collectivité elle-même, représentée par son exécutif.

Exemples :

- le maire pour une commune,
- le président pour une intercommunalité,
- le président du conseil départemental ou régional.

Cela signifie que la collectivité doit déterminer les finalités, les moyens et les mesures de protection des traitements.

## Le rôle du DPO en collectivité

Le **DPO** accompagne la collectivité dans sa conformité.

Il peut :

- sensibiliser les élus et agents,
- aider à tenir le registre des traitements,
- conseiller sur les projets numériques,
- vérifier les mentions d'information,
- accompagner les demandes des administrés,
- alerter sur les risques,
- échanger avec la CNIL si nécessaire.

Dans les petites collectivités, le DPO peut être mutualisé, par exemple via un centre de gestion ou une structure intercommunale.

## Le registre des traitements

La collectivité doit tenir un **registre des activités de traitement**.

Ce registre permet de savoir :

- quelles données sont traitées,
- pour quel service,
- pour quelle finalité,
- qui peut y accéder,
- quels prestataires interviennent,
- combien de temps les données sont conservées,
- quelles mesures de sécurité sont prévues.

Exemple de traitement à inscrire :

| Élément | Exemple |
| --- | --- |
| Traitement | Inscriptions scolaires |
| Finalité | Gérer l'inscription des enfants dans les écoles |
| Données | Identité de l'enfant, responsables légaux, adresse, informations nécessaires au suivi |
| Personnes concernées | Enfants et responsables légaux |
| Accès | Service scolaire, agents habilités |
| Conservation | Durée nécessaire à la gestion puis archivage selon les obligations applicables |
| Sécurité | Droits d'accès limités, sauvegardes, confidentialité des dossiers |

## Risques fréquents

Les collectivités doivent être vigilantes sur :

- la collecte excessive de données,
- la réutilisation de fichiers pour une autre finalité,
- l'accès trop large aux dossiers,
- les fichiers partagés sans contrôle,
- les prestataires mal encadrés,
- les caméras installées sans justification suffisante,
- les durées de conservation trop longues,
- la mauvaise gestion des demandes des administrés,
- les violations de données.

## Exemple de mauvaise pratique

Une mairie utilise le fichier des inscriptions scolaires pour envoyer une communication politique.

Problème : les données ont été collectées pour gérer la scolarité, pas pour faire de la communication politique.

Cela viole le principe de **finalité**.

## Bons réflexes

Pour une collectivité, les bons réflexes sont :

- désigner un DPO,
- tenir le registre à jour,
- limiter les données collectées,
- informer clairement les administrés,
- limiter les droits d'accès,
- encadrer les prestataires,
- sécuriser les logiciels métiers,
- prévoir une procédure en cas de violation de données,
- former les agents aux bons usages.

!!! tip "Réflexe AIS"
    En collectivité, un audit RGPD commence souvent par les applications métier : état civil, scolaire, RH, finances, action sociale, vidéosurveillance, portail citoyen, messagerie et sauvegardes.

## À retenir

- Les collectivités traitent de nombreuses données personnelles liées aux services publics.
- Certaines données peuvent concerner des publics vulnérables.
- Le DPO, le registre, la sécurité et l'information des administrés sont des points essentiels.
- Le RGPD renforce la confiance entre la collectivité et les citoyens.
