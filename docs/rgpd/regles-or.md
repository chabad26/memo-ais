# Les 8 règles d'or du RGPD

Les **8 règles d'or du RGPD** servent de repères simples pour vérifier qu'un traitement de données personnelles est correctement encadré.

Elles permettent de passer d'une idée vague de conformité à une méthode concrète : pourquoi je collecte, quoi, pendant combien de temps, avec quelle sécurité, et quels droits pour les personnes.

## 1. Licéité du traitement

Un traitement de données doit avoir une **base légale**.

Cela signifie qu'une organisation ne peut pas collecter ou utiliser des données personnelles simplement parce que c'est pratique. Elle doit pouvoir justifier juridiquement le traitement.

Bases légales fréquentes :

- consentement,
- contrat,
- obligation légale,
- mission d'intérêt public,
- intérêt légitime,
- sauvegarde des intérêts vitaux.

**Exemple :** une boutique en ligne traite l'adresse postale d'un client pour livrer une commande. La base légale est l'exécution du contrat.

## 2. Finalité du traitement

Les données doivent être collectées pour un objectif **déterminé, explicite et légitime**.

La finalité correspond à la raison précise pour laquelle les données sont utilisées.

**Exemple :** collecter une adresse e-mail pour envoyer une confirmation de commande est une finalité claire. La réutiliser ensuite pour envoyer de la publicité sans base adaptée peut poser problème.

## 3. Minimisation des données

Il faut collecter uniquement les données nécessaires à l'objectif prévu.

Cette règle évite les collectes excessives.

**Exemple :** pour créer un compte utilisateur sur un forum, une adresse e-mail et un mot de passe peuvent suffire. Demander une date de naissance, une adresse postale ou un numéro de téléphone n'est pas forcément justifié.

## 4. Protection particulière des données sensibles

Certaines données sont plus sensibles que d'autres et nécessitent une protection renforcée.

Exemples de données sensibles :

- données de santé,
- opinions politiques,
- convictions religieuses,
- appartenance syndicale,
- données biométriques,
- données génétiques,
- orientation sexuelle.

En principe, leur traitement est interdit, sauf exceptions prévues par le RGPD.

**Exemple :** une application médicale peut traiter des données de santé si cela est nécessaire au suivi du patient et si les garanties adaptées sont mises en place.

## 5. Conservation limitée des données

Les données ne doivent pas être conservées indéfiniment.

Une durée de conservation doit être définie selon la finalité du traitement.

À la fin de cette durée, les données doivent être :

- supprimées,
- anonymisées,
- ou archivées si une obligation le justifie.

**Exemple :** les logs de connexion d'une application peuvent être conservés pour la sécurité pendant une durée limitée, puis supprimés ou archivés selon la politique définie.

## 6. Sécurité des données

L'organisation doit protéger les données personnelles contre les accès non autorisés, la perte, la modification ou la destruction.

Mesures possibles :

- mots de passe robustes,
- MFA,
- chiffrement,
- sauvegardes,
- mises à jour,
- contrôle des droits,
- journalisation,
- segmentation réseau,
- procédure de gestion des incidents.

**Exemple :** une base client contenant des données personnelles doit être sauvegardée, protégée par des droits d'accès stricts et non accessible publiquement depuis Internet.

## 7. Transparence

Les personnes doivent être informées de manière claire sur l'utilisation de leurs données.

Elles doivent comprendre :

- qui traite les données,
- pourquoi les données sont collectées,
- combien de temps elles sont conservées,
- à qui elles peuvent être transmises,
- comment exercer leurs droits.

**Exemple :** un formulaire de contact doit indiquer à quoi servent les informations saisies et comment la personne peut demander l'accès, la rectification ou la suppression de ses données.

## 8. Droits des personnes

Le RGPD donne aux personnes des droits sur leurs données.

Principaux droits :

- droit à l'information,
- droit d'accès,
- droit de rectification,
- droit d'effacement,
- droit d'opposition,
- droit à la limitation,
- droit à la portabilité,
- droit de ne pas faire l'objet d'une décision entièrement automatisée dans certains cas.

**Exemple :** un utilisateur peut demander à une entreprise quelles données elle possède sur lui. Il peut aussi demander la correction d'une erreur ou la suppression de certaines données si les conditions sont réunies.

## Tableau récapitulatif

| Règle | Question à se poser | Exemple rapide |
| --- | --- | --- |
| Licéité | Ai-je une base légale ? | Adresse de livraison pour une commande |
| Finalité | Pourquoi les données sont-elles traitées ? | Envoyer une confirmation de commande |
| Minimisation | Ai-je vraiment besoin de cette donnée ? | Ne pas demander l'adresse postale pour une newsletter |
| Données sensibles | La donnée nécessite-t-elle une protection renforcée ? | Données de santé dans une application médicale |
| Conservation | Combien de temps garder les données ? | Supprimer les logs après la durée prévue |
| Sécurité | Les données sont-elles protégées ? | Chiffrement, sauvegarde, droits d'accès |
| Transparence | La personne comprend-elle le traitement ? | Mention d'information sur un formulaire |
| Droits | La personne peut-elle exercer ses droits ? | Demande d'accès ou de suppression |

## À retenir

- Les 8 règles d'or transforment le RGPD en réflexes pratiques.
- Un traitement doit être justifié, limité, expliqué, sécurisé et documenté.
- Pour un AIS, les règles les plus concrètes sont souvent la minimisation, la conservation, la sécurité, les logs et la gestion des droits.
