# Définition et enjeux basiques du RGPD

## Définition simple

Le **RGPD** est le règlement européen qui encadre la collecte, l'utilisation, la conservation et la protection des **données personnelles**.

Une donnée personnelle est une information qui permet d'identifier une personne directement ou indirectement.

Exemples :

- nom et prénom,
- adresse e-mail,
- numéro de téléphone,
- adresse IP,
- identifiant client,
- photo,
- numéro de sécurité sociale,
- données de localisation,
- données biométriques.

!!! note "Important"
    Le RGPD ne concerne pas seulement les fichiers informatiques. Un fichier papier contenant des informations personnelles est aussi concerné.

## Qu'est-ce qu'un traitement de données ?

Un **traitement** correspond à une action réalisée sur des données personnelles.

Exemples :

- collecter des données via un formulaire,
- enregistrer un compte utilisateur,
- envoyer une newsletter,
- stocker des logs de connexion,
- sauvegarder une base de données client,
- transmettre des données à un prestataire,
- supprimer ou archiver des données.

Pour un administrateur système ou réseau, le RGPD touche donc aussi les aspects techniques : comptes, droits d'accès, journaux, sauvegardes, supervision, chiffrement et sécurité des serveurs.

## Objectifs principaux

Le RGPD poursuit plusieurs objectifs :

- protéger les personnes,
- donner plus de contrôle aux citoyens sur leurs données,
- responsabiliser les organisations,
- harmoniser les règles dans l'Union européenne,
- renforcer la confiance dans les usages numériques.

## Risques liés aux données personnelles

Les données personnelles peuvent sembler banales lorsqu'elles sont prises séparément.

Exemple : une adresse e-mail, une adresse IP ou un historique de connexion ne paraît pas toujours sensible au premier regard.

Mais lorsqu'on croise plusieurs données, il devient possible de mieux connaître une personne, ses habitudes, ses déplacements, ses centres d'intérêt ou son comportement en ligne.

## Traçage possible

Le **traçage** consiste à suivre ou reconstituer l'activité d'une personne à partir de données collectées.

Exemples de données utilisées pour tracer une personne :

- adresse IP,
- cookies,
- historique de navigation,
- logs de connexion,
- données de localisation,
- identifiants publicitaires,
- adresse e-mail utilisée sur plusieurs services,
- données issues d'applications mobiles,
- achats ou interactions sur une plateforme.

Le traçage peut servir à des usages légitimes, comme la sécurité, la lutte contre la fraude ou le diagnostic technique.

Mais il peut aussi devenir problématique s'il permet :

- de surveiller une personne sans qu'elle le comprenne,
- de créer un profil détaillé de ses habitudes,
- de suivre ses déplacements,
- de déduire des informations sensibles,
- d'influencer ses choix par de la publicité ciblée,
- de partager ses données avec trop de prestataires,
- de conserver des traces trop longtemps.

!!! warning "Point de vigilance"
    Une donnée technique peut devenir une donnée personnelle si elle permet d'identifier directement ou indirectement une personne. Par exemple, une adresse IP ou un log de connexion peut être concerné par le RGPD.

## Risques concrets pour les personnes

Une mauvaise gestion des données personnelles peut entraîner :

- atteinte à la vie privée,
- usurpation d'identité,
- hameçonnage ciblé,
- discrimination,
- surveillance abusive,
- perte de contrôle sur ses informations,
- exposition de données sensibles,
- préjudice financier ou réputationnel.

Pour une organisation, ces risques peuvent aussi provoquer :

- perte de confiance,
- incident de sécurité,
- plainte auprès de la CNIL,
- sanction,
- obligation d'informer les personnes concernées,
- impact sur l'image de l'entreprise ou de l'administration.

## Les grands principes

### Finalité

Les données doivent être collectées pour un objectif clair, précis et légitime.

Exemple : collecter une adresse e-mail pour créer un compte utilisateur.

### Minimisation

On ne collecte que les données réellement nécessaires.

Exemple : pour créer un compte de forum, demander une adresse e-mail peut être utile ; demander le numéro de sécurité sociale serait excessif.

### Transparence

La personne doit comprendre pourquoi ses données sont collectées, comment elles seront utilisées et pendant combien de temps elles seront conservées.

### Durée de conservation limitée

Les données ne doivent pas être gardées indéfiniment.

Exemple : un ancien compte client inactif depuis plusieurs années doit être supprimé ou archivé selon une règle claire.

### Sécurité

L'organisation doit protéger les données contre l'accès non autorisé, la perte, la destruction ou la modification.

Exemples de mesures :

- mots de passe robustes,
- MFA,
- sauvegardes,
- chiffrement,
- contrôle des droits,
- mises à jour,
- journalisation,
- cloisonnement réseau.

## Les droits des personnes

Le RGPD donne plusieurs droits aux personnes concernées :

- droit à l'information,
- droit d'accès,
- droit de rectification,
- droit d'effacement,
- droit d'opposition,
- droit à la limitation du traitement,
- droit à la portabilité,
- droit de retirer son consentement quand le traitement repose sur le consentement.

## Enjeux basiques pour une organisation

Une organisation doit pouvoir répondre à trois questions simples :

1. Quelles données personnelles sont traitées ?
2. Pourquoi ces données sont-elles traitées ?
3. Comment sont-elles protégées ?

Si une organisation ne sait pas répondre, elle ne maîtrise pas vraiment sa conformité.

## À retenir

- Le RGPD protège les données personnelles des personnes physiques.
- Toute collecte ou utilisation de données doit avoir un objectif clair.
- Le croisement de données peut permettre de tracer ou profiler une personne.
- Les données doivent être limitées, sécurisées et conservées seulement le temps nécessaire.
- Le RGPD n'est pas seulement juridique : il a des conséquences directes sur l'administration système, réseau et sécurité.
